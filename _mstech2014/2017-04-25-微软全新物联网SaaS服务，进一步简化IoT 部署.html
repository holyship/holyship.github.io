微软推出全新物联网SaaS服务 Microsoft IoT Central，进一步简化IoT 部署

物联网（IoT）的应用和推广正在迅速发展成为各行各业开展业务的一个核心组成部分。互联网、移动设备和云计算在过去相当的一段时间推动了数字化转型的进程，而物联网将是下一个重要的催化剂。

不过，对很多希望保持竞争优势的企业来说，虽然物联网能带来非常多新的优势，但是在如何利用物联网的问题上，他们也碰到了很多挑战——物联网解决方案的技术壁垒仍然较高，而技术能力的缺乏让很多用户仍然难以利用物联网来进行转型和创新。为了帮助解决这些难题并简化物联网的部署，微软很高兴地在本周对外宣布了一些新技术和新方案。

正式对外公布全新SaaS（软件即服务）物联网方案：Microsoft IoT Central，进一步简化物联网部署

微软正式对外公布了全新的SaaS（软件即服务）物联网方案：Microsoft IoT Central。新方案将进一步降低用户打造物联网方案的复杂度。Microsoft IoT Central是一个完全托管的SaaS（软件即服务）产品，客户和合作伙伴无需云计算专业知识，就能够获得强大的物联网方案。

基于微软智能云服务Azure的Microsoft IoT Central进一步简化了开发过程，使用户能轻松入门，迅速利用物联网，搭上数字化转型的快班车。

Microsoft IoT Central将与微软现有的PaaS（平台即服务）方案 Azure IoT 套件一起面向广大的物联网用户。现有的Azure IoT套件将继续适用于需要深度定制和全面掌控的用户场景。全新公布的Microsoft IoT Central SaaS则将最大限度地降低创建物联网解决方案的复杂度，帮助制造业用户极大地加速创新，敏锐抓住市场机遇，向市场快速投放新产品，带来新收入，并为最终用户提供更好的体验。

Microsoft IoT Central将在未来几个月中逐步推向市场，有兴趣的朋友可以访问Microsoft IoT Central.com进行注册及查看更多更新内容。

“智能工厂”Microsoft Azure IoT套件

针对具有专业云计算知识的客户和合作伙伴，微软正在继续投资我们现有的PaaS 产品：微软 Azure IoT Suite物联网套件。本周在汉诺威工业展上，微软将推出一个新的预配置Azure IoT Suite物联网套件：智能工厂。新的智能工厂套件将加速客户工业革命4.0之旅，便于将内部OPC UA和OPC传统设备连接到微软智能云，帮助提高运营效率和洞察力。同时，它还能帮助客户从云中安全地浏览和配置工厂设备。

我们正在与几个OPCUA领域提供交钥匙网关解决方案的业界领袖一起，内置网关和Azure连接的方案，对用户实现几乎零配置的连接使用。这些合作伙伴包括Unified Automation，Softing和Hewlett Packard Enterprise。

希望更多了解智能工厂方案为客户和合作伙伴提供的价值，请登陆Azure IoT 套件主页、技术博客帖、也可查看智能工厂六步框架白皮书和演示。

Microsoft Azure Time Series Insights

物联网设备和传感器大都基于时间的方式发送数据，而方案常常要求能发现数据中的异常和趋势。但到目前为止，很多企业不得不聘请合作伙伴或专家来满足这一需求。

微软宣布了一个新的服务:Azure Time Series Insights，一个完全托管的分析、存储和可视化服务。这个服务将帮助物联网用户从成千上万的事件中即时和互动地进行探索和分析。

Azure Time Series Insights提供了跨各种事件源的数据的全局视图，因此用户可以快速验证其物联网解决方案，并避免关键任务设备的宕机。它帮助用户发现隐藏的趋势、发现异常现象，并在接近实时的情况下分析根本原因。操作体验简单直观，无需编写代码。此外，它还提供了丰富的API，使用户能够将其强大的功能集成到现有的工作流和应用程序中。

Azure Time Series Insights服务目前还处于预览状态。Microsoft IoT Central和“智能工厂”Azure IoT Suite物联网套件预配置解决方案都将利用全新的Azure Time Series Insights服务。

目前已经有一些客户和合作伙伴，如蒂森克虏伯电梯和Codit开始利用Time Series Insights来降低成本和满足客户的需求。要了解更多，请阅读我们的技术博客，并注册试用 AzureTime Series Insights 预览。

在边缘设备上的微软 Azure Stream Analytics流分析服务

尽管很多开发者通过把物联网设备连接到云上获利，但有些场景还是需要边缘智能来获得最佳的物联网体验。在2016年，微软推出了 Azure 物联网网关SDK，使开发者和ISV 能够轻松地构建和部署适合其特定场景的网关智能化。

微软宣布在边缘设备上的 Azure Stream Analytics流分析服务预览版，这是 Azure 流分析服务的一个新功能，它将基于云服务的流处理技术延伸到设备端。

在边缘设备上的 Azure流分析服务和其云端服务具有统一的云管理经验。在云计算的连接能力有限或不稳定的情况下，使用这种流分析服务的方式，使得用户仍然能够获得对业务至关重要快速洞察力并主动采取行动。

要了解更多，请阅读我们的技术博客文章，并注册试用Azure 流分析服务预览。

Microsoft Azure IoT Hub设备配置服务

微软致力于减少打造物联网项目所需的复杂性和时间。新的Azure IoT Hub设备配置服务（Device Provisioning）简化了注册流程，使用户能够在节省时间和资源的同时，以安全和可伸缩的方式自动注册和配置设备到IoT Hub服务。设备配置服务将与现有IoT Hub支持的设备共同工作。

对于运行 Windows 10 IoT操作系统的设备，通过OEM在设备中加入的客户端应用程序，该类设备将能更简便地连接到设备配置服务。使用Windows10 IoT，用户可以获得零触碰的配置体验，在连接新的物联网设备到 Azure 服务时，消除配置和资源调配麻烦。

如果结合 Windows 10 IoT 对 Azure IoT Hub服务设备管理的支持，通过利用设备搬迁、所有权转移、安全设备管理，和设备淘汰管理等功能，整个设备生命周期管理将被极大简化。您可以通过访问Azure IoT设备管理和阅读我们的技术博客，了解更多关于 Windows物联网设备配置和设备管理细节的信息。

与硬件合作伙伴一起，走向物联网的安全高地

微软将一如既往恪守保障物联网安全的承诺。我们宣布Azure物联网对一批硬件和标准的广泛支持，继续加强我们对物联网安全的承诺。

Azure物联网现在支持行业安全标准DeviceIdentity Composition Engine（DICE）和Hardware Security Module （HSM）。DICE标准使制造商能够使用硅闸门创建基于硬件的设备标识，使安全硬件从一开始就成为新设备基因的一部分。HSM是保护设备标识并提供高级功能（如基于硬件的设备认证和零触点配置）的核心安全技术。

微软还宣布与Micron、STMicro合作，支持利用HSM和DICE安全技术，同时和 Spyrus合作支持HSM作为SD和USB存储设备的一部分。

在本周的汉诺威工业展上，微软将展示如何使用设备识别和认证模型来保护大型和小型的物联网设备。一旦设备启用自动注册和分配给Azure IoT Hub服务，就可以实现身份验证和配置过程的自动化，节省大量的物联网部署时间。活动将展示来自Micron和STMicro的硬件产品。
