本文作者：Mike Neil，微软Azure基础设施与管理副总裁

 

通过Azure Stack之前的几个技术预览版本，我们看到了众多客户和合作伙伴对于Azure Stack的兴奋之情。事实上，在本周举办的微软Inspire大会上，我们与数千名合作伙伴进行了交流，他们同样为“One Azure Ecosystem”方案所带来的全新商业机遇倍感振奋。它可以帮助客户把在Azure上的现有投资延伸到Azure Stack，为混合云环境解带来新的可能性。基于此，微软今天宣布：

 

  • Azure Stack集成系统开放预定：我们已经向硬件合作伙伴交付了Azure Stack软件，从而能够开始对其集成系统进行认证，首批系统将于9月开始出货。客户可从我们的硬件合作伙伴Dell EMC、HPE和联想那里订购集成系统。   

  • Azure Stack软件的定价及可用性：Azure Stack提供按需付费和按容量付费的定价模式，客户可根据这些信息制定采购计划。 

  • Azure Stack开发工具包（ASDK）开放下载：用于测试用途的ASDK已经开放网上下载。该工具包针对用于试用的免费单服务器部署选项，用户可以借助其开发和验证自己的应用，以用于集成系统上的部署。

 

Azure Stack的承诺

作为微软智能云Azure的扩展，Azure Stack提供了真正一致的混合云平台。这样的一致性让混合云变得不再复杂，并且能够帮助客户最大化云及本地环境的投资回报：用户能够使用完全一致的手段——包括同样的API、同样的DevOps工具、同样的门户网站——来开发和部署应用，从而提高开发者的生产力；用户能够基于Azure Marketplace应用组件更快速地开发云应用；一致性确保了企业资源的可转移性，从而让客户能够更加安心地对人力和流程进行投入；Azure Stack能够在本地运行一致的Azure服务，从而让用户能够更加灵活的部署应用和工作负载。基于集成系统的交付模式确保客户可以聚焦于对业务更为关键的环节（例如你的应用），同时也确保微软能够快速地向客户交付Azure创新功能。

 

在首次发布的版本中，Azure Stack包含了一套核心Azure服务、DevOps工具和Azure Marketplace内容，所有这些都将通过一套集成系统进行交付。


混合应用场景带来应用创新    

针对企业服务其客户，以及企业内部的业务应用，微软智能云Azure和Azure  Stack带来了全新的应用场景：

  • 边缘和分离解决方案：借助相同的应用逻辑，客户可通过Azure Stack在本地进行数据处理，之后将其转移到Azure进行进一步分析，以满足延迟和连接要求。我们发现有很多的环境都对这样的边缘应用场景有着需求，包括工厂车间、航船和矿井等。

  • 提供满足多种法规要求的云应用：用户可以在微软智能云Azure中开发和部署应用，并灵活地在Azure Stack上进行本地部署，以满足监管或政策要求，且无需更改代码。很多客户正寻求在不同环境下部署相同应用（如：一个全球性的审计或财务报告应用软件）的可能性，以满足业务和技术要求。微软智能云Azure能够满足大多数要求，而Azure Stack则能够实现多地的本地部署。盛宝银行（Saxo Bank）就是一个很好的例子，目前这家企业正在计划充分利用Azure Stack实现灵活部署。  

  •  本地云应用模式：用户可以借助Azure web和移动服务、容器、无服务器和微服务架构来更新和扩展现有应用或开发新应用。在基于微软智能云Azure的云上，和基于Azure Stack的本地环境中，用户可以使用一致的DevOps流程。微软看到了用户对对应用现代化的广泛兴趣，这其中也包括了核心的关键业务应用。日本三井物产公司（Mitsui）就是一个很好的例子，这家公司目前正使用Azure Stack和微软智能云Azure规划自己的应用现代化路线图。 

 

跨微软智能云Azure和Azure Stack的生态系统解决方案 

通过利用完善的Azure生态系统，用户可以更加快速地适应Azure Stack：

  • 我们的目标是，确保大多数通过Azure认证的ISV应用和服务也能够支持Azure Stack。包括Bitnami、Docker、Kemp Technologies、Pivotal Cloud Foundry、Red Hat Enterprise Linux和SUSE Linux在内的诸多ISV正努力在Azure Stack上提供他们的解决方案。

  • 在Azure Stack交付和运营模式上，用户可以选择全托管服务。包括Avanade、Daisy、Evry、Rackspace和Tieto在内的诸多合作伙伴正在为Azure和Azure Stack提供托管服务。这些合作伙伴一直通过云解决方案提供商（CSP）计划为Azure提供托管服务，目前他们正在拓展其产业及业务范围，以期能够提供混合云解决方案。

  • 系统集成商则提供了深度Azure技能、相关领域和行业知识以及流程专业知识（例如DevOps），从而帮助客户加速器应用现代化的进度。普华永道（PwC）就是一个很好的例子，目前这家公司正把其咨询业务扩展到Azure和Azure Stack。 

 

集成系统开放预订，并提供用于试用的免费单服务器工具包    

 Azure Stack提供两个部署选项：

  • Azure Stack集成系统——这是用于生产的多服务器系统，旨在快速投入使用。根据硬件偏好，客户可以选择Dell EMC、HPE和联想（思科和华为也将在晚些时候加入该计划）的集成系统。客户现在就可以查找到这些已认证的硬件解决方案，并联系这些硬件合作伙伴来订购集成系统。且无论客户选择哪家硬件合作伙伴的产品，这些系统都可以立即投入使用，并提供一致的、端到端的客户支持。这些集成系统将率先在46个国家开放购买，覆盖了全球的主要市场。

  • Azure Stack开发工具包（ASDK）——ASDK是一个免费的单服务器部署工具，用于试用和概念验证。ASDK即日起可以在网络上下载，用户可以借助该工具包开发原型应用。在ASDK版本中，门户网站、Azure服务、DevOps工具和Marketplace内容与集成系统版本是一致的。因此，使用ASDK版本开发的应用可以部署到多服务器系统上。

 

结语

 

作为微软智能云Azure的扩展，Azure Stack将在初始版本发布后继续保持更新，持续地为客户提供创新服务，从而为客户交付更为丰富的混合云应用使用场景，并对Azure Stack的基础设施进行提升。我们还将继续扩大微软智能云Azure的生态系统，为客户带来更多的选择和灵活性。

 

期待各位能够借助AzureStack进行更多创新！
