移动互联时代，无论衣食住行，人民的生活中方方面面的需求都可以通过智能手机、平板电脑等移动设备，以及五花八门的应用来满足。

那么，在向着「服务型」转型的今天，政府机构和教育机构，能否“以人民之名，为人民服务”，让人民享受到移动化所带来的便利呢？

有网站，有网上办事大厅，还要怎么「移动」？

几乎所有政府部门、教育机构、企业事业单位都已经「上网」，各地都在建设网上办事大厅，且很多机构不仅有自己的网站，还会通过网站发布各类信息和资源。但移动化之路，这才是迈出了第一步，后面还有很多问题等着解决：

  • 老百姓希望随时访问政府网站查阅最新公告和通知，可是用手机访问后却无奈地发现，整个版面乱做一团，文字和图片密密麻麻挤在一起，根本看不清，通过页面办理业务就更困难了……

  • 学校正在推行电子化教学，要求学生将电子版作业提交到班级门户网站，孩子在平板上写完的作业打算提交，可班级门户里上传文档的按钮在平板上始终无法显示，只能借助电脑上传……

  • 企业通过自己网站发布最新的产品和服务信息，然而所发布的内容用了很长时间也没更新到移动端网站系统，移动端用户只能看到「过时」的信息，极大影响了时效性和网站访问体验……

用户经常需要访问不同的网站和 Web 应用。然而最悲剧的地方在于，因为一些历史原因，很多网站在开发时主要针对了传统的台式和笔记本电脑（也就是桌面端），并没有针对移动设备的特点进行适配。

这就导致很多网站在使用电脑访问的时候完全没有任何问题，但如果用手机或者平板来访问，轻则版面错乱或文字太小看不清，重则重要功能完全无法使用。

为了解决这种问题，很多人考虑过各种方法。例如：

  • 针对移动设备单独开发一个网站。然而这种方式需要不菲的人力物力和时间投入，而且两个网站的维护和内容更新发布也会造成巨大的工作负担。

  • 或者针对不同移动平台开发原生应用，以应用的形式提供原本通过网站即可提供的服务和功能。然而这种做法门槛高，耗时久，后期维护和更新负担重。

  • 或者使用「响应式布局」网页设计，自动针对设备特征呈现出最优化的页面版式。但这种方式对网页设计技术提出了更高要求，开发周期长，升级更新成本高。

上述所有方式，费时费力费钱不说，效果也未必非常好。

但是，如果有一种新方法，无须对原本的网站进行太多改动，即可自动适配不同设备？

PC站群及门户移动化解决方案，解决移动适配大问题

“PC 站群及门户移动化解决方案”，是微软合作伙伴“云适配”基于其前端适配技术以及微软 Azure云计算平台打造的解决方案，可迅速为政府和企业构建安全、便捷、跨平台的 HTML5 移动应用系统，帮助各类机构实现跨屏营销和政务教育信息移动化战略。

使用该解决方案后，网站只需在自己的网页中插入相应代码，无须其他任何改动，即可取代以往需要繁琐的编程开发和数据迁移等工作，直接在原网址的基础上为网站提供针对移动端进行优化的版本。基于 Azure 公有云平台的“PC 站群及门户移动化解决方案”，全面解决了手机和其他移动终端屏幕尺寸碎片化、屏幕分辨率不统一、浏览器分化这三大移动终端适配难题，可自动识别适配不同终端和系统，适配全球超过 5000 种移动设备！ 


Microsoft Azure，让桌面端网站「轻装上阵」

作为该解决方案的核心，其云端适配平台完全基于 Azure 智能云构建，通过适配平台对统对桌面端网站的目标页面抓取和分析相关数据，在云端计算出设备的网页最佳展现方式。此外，借助全球独创的图片压缩技术，可平均节省图片流量 80% 以上，下载速度提高 5 倍以上。

将云端适配平台托管在 Microsoft Azure 上，主要出于下列几方面考虑：

  • 自动伸缩更敏捷：Azure 灵活的自动扩展功能可对访问量变化做出快速响应。流量大时 Azure 自行分配更多资源，适应大流量；流量变小后将资源回收。按用量付费，保障服务质量的同时将成本降至最低。

  • 网络资源更充裕：该解决方案的适配过程会频繁通过网络传输数据，网速不畅将严重影响客户网站的打开效果和流畅程度。Azure 数据中心通过 BGP 方式直连多家主流运营商的省级核心网络节点，可为用户提供高速稳定的网络访问体验。

  • 稳定可靠有保障：服务永远在线、永远可用对云适配来说非常重要。Azure 的企业级服务等级协议（SLA）为该解决方案提供了一套可靠的基础设施，曾经只有大型企业才能用得上的服务也可以被创业公司轻松使用。

目前已有大量的政府机构和企业成功应用该解决方案，快速将桌面端网站适配到各种移动终端设备，不仅显著地提高了网站的用户体验以及销售转化率，而且大幅节省了开发、维护以及推广移动网站的费用。
