随着业务的快速发展，对IT的稳定性，业务连续性要求越来越高。IT面临的问题：您的数据安全吗？您的系统可靠吗？什么样的方案才是低成本和高可靠的容灾与备份的解决方案？

杭州华三通信技术有限公司（华三通信 H3C），作为致力于IT基础架构产品和方案的传统厂商，率先意识到需要快速融入到云计算的技术更新浪潮，以保持对新技术的敏感性和IT 系统架构的先进性。微软Azure的容灾备份解决方案，以其灵活、快速、可靠、成本低的特性，成为了H3C的最佳选择。

叶巍 IT部总监 华三通信



应用背景


H3C主要提供IT 基础架构产品及方案的研究、开发、生产、销售及服务。H3C在中国设有38个分支机构，目前公司有员工5000人。为了提高灵活性，以及快速响应的要求，H3C将现有数据中心内的Hosting环境迁移到Azure上，包括在线培训系统，移动应用平台以及企业微信平台。同时对数据的安全性和业务连续性也提出了新的要求。

IT挑战


传统的灾备方案面临如下挑战：

1. 建设成本高，一般灾备数据中心投资几百万起

2. 建设周期长，维护成本高

3. 技术复杂，如稳定的电源，散热，网络等等

鉴于投资和维护成本巨大，很多用户的数据备份多采用冷备方式。但当灾难发生时，存在业务恢复慢，甚至有数据丢失的风险。

解决方案



图：本地+Azure双活机制架构图

1. Azure上海数据中心和H3C杭州内部数据中心的HA双活

  • 使用Azure Traffic Manager进行DNS级别切换，双站点间基于Failover负载均衡模式进行切换，RTO <= 1h，并且切换为自动完成，RPO取决于后台向前段的发布频率，满足了业务容灾需求。

  • 使用Failover模式，平日只需少量Azure服务器应对突发状况即可，降低了成本，当灾难发生时，可以动态开启更多节点。

  • 使用 Azure VPN gateway连接Azure数据中心及H3C内部数据中心，保证网站前台和后端之间的高速连接，另外专有网络也保证了数据传输的安全性。

2. Azure 负载均衡和可用性集进一步保证了Azure 站点服务可用性

  • 三台Web服务器（Windows Server 2012 R2 Datacenter）在一个可用性集中, 通过负载均衡(SLB)对外提供服务，在一个数据中心内部形成冗余，进一步提高SLA。

  • 两台Download服务器（Windows Server 2012 R2 Datacenter）在一个可用性集中, 通过负载均衡(SLB)对外提供服务，在一个数据中心内部形成冗余，使用单独存储账号中的共享存储避免单台共享服务器的单点故障，进一步提高SLA。

3. Azure VM 备份和数据备份进一步保护Azure服务器和应用

  • 通过Azure Backup Service 每天备份 Web 服务器和Download 服务器，在服务器误删误改时，可以快速恢复之前的备份点,快速恢复Azure VM。

  • 通过System Center DPM 备份Azure VM server中的应用数据，保留多个可用复本，以便恢复之前版本数据，满足业务管理需要。

客户收益


  • 低成本的投入，建立了高完善的数据灾备解决方案。

  • 通过引入Azure数据中心，将传统的千万级别的灾备中心投入成本降低到十万级别。

  • Azure灾备方案带来了更多的灵活性，弹性，以及可扩展的灾备解决方案。
