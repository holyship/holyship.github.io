编者按：本文由Betsey LIU原创。SVG是一个大家熟悉又陌生的领域，你一定在工作学习中或多或少的接触到SVG，但是对于给SVG元素描边这件事情，你是否完全掌握呢？

    在做声享项目的时候，遇到了一个比较棘手的问题：有用户反馈说希望可以给形状加边框。

    声享在实现改变形状元素样式的方法，是给 SVG 元素外面包裹着的 div 元素增加样式。     比如说当 SVG 元素的 width:100%;height:100%的时候，调整外面包裹的div的宽度和高度就可以修改 SVG 的宽高。 同样的来说，修改 div color属性，同时将svg中的样式写成fill:currentColor，就可以继承父元素的color，达到修改形状颜色的目的 。     这样做可以很方便地修改 SVG 的样式，但是当面对需要给形状增加边框这个需求的时候，单纯地为父元素 div 增加 border是不行的，会成为下图的这个效果：

    想要给SVG形状增加边框就必须在SVG上进行修改。     当对 SVG 有了一定的研究后，多数人会进行尝试，为 SVG 增加stroke和stroke-width这两个属性。

    但当为 SVG 增加了这两个属性之后，新的问题又出现了： 增加的边框跨在了 SVG 形状上，导致有50%的边框超出了SVG形状的最大边缘。下面的这个图中，为 SVG 增加了边框，为了更好地展现边框是跨在形状上，我另外为边框增加了0.5的透明度，style="stroke:orange;stroke-width:5;stroke-opacity:0.5"。

    可怜的是，SVG无法决定增加的边框的位置，没有inside这种参数选项。    

    所以，为了解决这个问题，需要另辟蹊径。比如说，我们可以对SVG的图片进行裁剪，使用clip-path这个属性。

    当在 SVG的<defs>里面定义一个<clipPath>，就为SVG增加了一个裁剪路径。当形状应用这个裁剪路径的时候，我们只能够看到裁剪路径所包含的那部分形状，而裁剪路径以外的形状就被隐藏掉了。实际上就相当于，在这个形状上增加一个镂空的蒙层。

    我们需要为每一个裁剪路径指定一个唯一的id。比如我们需要为这个心形增加一个和它一样形状的裁剪路径就可以这样定义:

<div class="heart-wrapper">
  <svg xmlns="http://www.w3.org/2000/svg" xmlns:xlink="http://www.w3.org/1999/xlink" version="1.1" id="heart" width="100%" height="100%" viewBox="0 0 60 60">
    // 定义的裁剪路径
    <defs>
      <clipPath id="heartClip">
        // 这个裁剪路径是和下面的心形一样的
        <path d="M28.487,10.847C21.13-6.364,0.18-2.348,0.08,17.628C0,33.538,27.699,46.784,28.531,49.636 C29.285,46.675,57,33.785,56.92,17.509C56.823-2.517,35.506-5.678,28.487,10.847z">
      </clipPath>
    </defs>
    <path fill="#E87A90" style="stroke:orange;stroke-width:5" clip-path="url(#heartClip)" d="M28.487,10.847C21.13-6.364,0.18-2.348,0.08,17.628C0,33.538,27.699,46.784,28.531,49.636 C29.285,46.675,57,33.785,56.92,17.509C56.823-2.517,35.506-5.678,28.487,10.847z">
    </path>
  </svg>
</div>

    同时在下面的心形的路径里面增加clip-path="url(#heartClip)"，这样这个裁剪路径就被应用到了心形上。

    可以看到，有一半的边框被裁剪路径裁剪掉了，实际上我们的边框宽度仅为stroke-width的一半。当我们需要增加边框宽度的时候，我们不会改变形状的大小，边框宽度向内增加了，是不是很神奇！？

    这种做法，对于希望为形状的所有区域都增加边框的时候，可以很完美地实现，但是，有些时候，我们只需要为形状的外部增加边框，对于形状内部的部分，我们希望保持原样。

    动动脑筋，我们可以在当前的形状上面增加一个透明不可见的形状，透明形状仅为原形状的外轮廓路径，当我们为这个透明形状增加边框的时候，实际上展现的效果就是为这个形状增加了外围边框。

<div class="youtube-wrapper">

    <defs>
      <clipPath id="youtubeClip">
      <path d="M728,544 C732.418278,544 736,540.418278 736,536 C736,531.581722 732.418278,528 728,528 C723.581722,528 720,531.581722 720,536 C720,540.418278 723.581722,544 728,544 L728,544 Z">
        </clipPath>
    </defs>
    <g fill="none" fill-rule="evenodd" id="Icons with numbers" stroke="none" stroke-width="1">
      <g fill="red" id="youtube" transform="translate(-720.000000, -528.000000)">

        </path>
      </g>
       <g fill="transparent" id="transparentYoutube" clip-path="url(#youtubeClip)" style="stroke:orange;stroke-width:2" transform="translate(-720.000000, -528.000000)">
        <path d="M728,544 C732.418278,544 736,540.418278 736,536 C736,531.581722 732.418278,528 728,528 C723.581722,528 720,531.581722 720,536 C720,540.418278 723.581722,544 728,544 L728,544 Z">
        </path>

</div>
