TL;DR: 如果你熟悉Flexbox, Grid对你来说也不会陌生。Rachel Andrew 维护了一个 CSS Grid 主题网站可以帮助你入门。Grid现在在Google Chrome已经得到了支持。

Flexbox? Grid?

过去几年，CSS Flexbox已经被广泛使用而且得到了大多数浏览器的支持 （难道你还不肯抛弃IE9以及以下版本的浏览器？！）Flexbox让复杂的布局变得容易，就像实现元素之间的等距离间距，上下两端对齐布局或者是CSS的经典问题—垂直居中布局。

可惜的是，我们需要考虑两个维度的布局。flex布局无法自动调整元素的大小，仅仅使用flexbox无法同时实现布局上水平和垂直上的要求。因此，CSS Grid横空出世了。

CSS Grid仍处于开发阶段。主流浏览器已经投入超过5年时间来实现它，并额外花了大量时间去解决交互问题，以避免像Flexbox当初问世之后问题多多的窘况。所以如果你在Chrome使用Gird布局，你会在Firefox和Safari看到一样的效果。在本文写作期间，Microsoft的Edg浏览器支持的Gird还不是最新的（IE11也有相同的问题）。而关于规范实现的更新，Edge 还在考虑当中。

尽管在概念和语法上Flexbox和Grid有点相似，但是别把他们看作两个完全一样的布局技术。Grid针对二维布局，而Flexbox是针对一维的。你完全可以两种布局的协同使用来共同完成某种布局效果。

定义一个栅格

为了熟悉Gird的每一个属性，我衷心推荐Rachel Andrew的Grid By Example或者 CSS Tricks'的Cheat Sheet。如果对Flexbox熟悉，Grid的许多属性对你来说也不会陌生。

让我们先看一下标准的12列grid布局。一个典型的12列布局就是可以用2，3，4或者6来进行等分12列，因此对许多布局设计都非常有帮助。我们可以试着实现下面的这种布局：

首先是HTML部分:

<!DOCTYPE html>
<body>
  <header></header>
  <nav></nav>
  <main></main>
  <footer></footer>
</body>

在我们的样式表，我们可以将整个body元素占满整个视图，并且用它作为一个grid容器

html, body {
  width: 100vw;
  min-height: 100vh;
  margin: 0;
  padding: 0;
}
body {
  display: grid;
}

现在我们就开始使用CSS Grid吧，万岁！

下一步就是实现grid中的rows和columns，我们本可以在样式中将12列都实现，但是因为并没用到所有列，这么做只会让CSS更乱。为了简单起见，实现的布局如下：

header和footer在宽度上是自适应的，content部分则在宽度和高度上都实现自适应，nav部分也一样实现两个维度的自适应，但是我们希望它最小宽度为200px（为什么？当然只是为了展示Grid的特性）。

在CSS Grid中，colums和rows的集合称为轨道（tracks）。我们首先来定义水平方向的轨道：

body {
  display: grid;
  grid-template-rows: 150px auto 100px;
}

grid-template-rows 属性接收一系列的参数来设置独立的行。在本例中，我们把第一行设置为150px的高度，最后一行设置为100px的高度。中间的一行设置为auto，这代表它会根据grid items(即grid container的子元素)来调整自身的高度。因为body元素在这里是占满整个视图的，该轨道包含content的部分（上图中的黄色区域）会至少占满整个可用的空间，但是如有必要，比如允许文档滚动的情况，它也可以进行进一步地扩展。

而垂直方向，我们想要实现一种更加动态的方法：nav和content部分能够实现自适应，但是nav不能小于200px的宽度，content还不能大于nav部分。在flexbox布局我们会使用flex-grow和flex-shrink，但是在Gird的实现有些不一样：

body {
  display: grid;
  grid-template-rows: 150px auto 100px;
  grid-template-columns: minmax(200px, 3fr) 9fr;
}

我们定义了两列。第一列使用了minmax()函数，它接收两个值：轨道的最小值和最大值（就像min-width和max-width的组合使用）。最小值我们设置为上面提到的200px。最大值则是3fr。fr是grid中特定的单元，它允许你对grid元素分配可用的空间(fr可能是"fraction unit"的缩写，但也可能在不久后表示"free unit")。我们这里配置的是两列都可以自适应充满整个视图，但是content列永远是nav列的三倍宽（而nav列的宽度至少大于200px）。

虽然现在例子的grid元素的位置还不完全正确，但是行和列的大小都已经满足设计要求了。

安置元素的位置

Grid最强大的一个特性是可以不必按照DOM的顺序决定元素的位置。（尽管如此，因为屏幕阅读器是根据DOM的顺序来的，我们还是强烈建议你多方面考虑来确定元素的书写位置）。如果没有手动设置元素的位置，那么元素就会按照DOM的顺序从左到右，从上到下的排列来放置。每个元素占据一个单元(cell)。这个顺序可以通过使用grid-auto-flow来修改。

所以，现在要怎么安置元素的顺序？最简单的方法就是通过定义元素占据哪一行哪一列来实现。Gird提供了两个语法：第一个语法是定义开始和结束的位置，第二种是定义开始的位置以及要跨越的距离。

header {
  grid-column: 1 / 3;
}
nav {
  grid-row: 2 / span 2;
}

在上面的代码中，我们让header从第一列开始，在第三列前面结束。nav则是从第二行开始，并且总共跨越两行的距离。

技术上，我们实现了我们想要的布局。但是我会介绍几个Grid的便捷特性，让安置元素更容易些。在Gird中，你可以用中括号给轨道命名然后指定它们的位置：

body {
  display: grid;
  grid-template-rows: 150px [nav-start] auto 100px [nav-end];
  grid-template-columns: [header-start] minmax(200px, 3fr) 9fr [header-end];
}
header {
  grid-column: header-start / header-end;
}
nav {
  grid-row: nav-start / nav-end;
}

这段代码与前面的代码生成的布局一样。

更厉害的是，你甚至可以用grid-area给每个区域命名：

body {
  display: grid;
  grid-template-rows: 150px auto 100px;
  grid-template-columns: minmax(200px, 3fr) 9fr;
  grid-template-areas: "header header"
                      "nav content"
                      "nav footer";
}
header {
  grid-area: header;
}
nav {
  grid-area: nav;
}

grid-template-areas 接收一个用空格进行分隔的字符串，允许开发者对每个区域单元进行命名。如果两个相邻的单元同名，它们会合并成一个区域单位。这样对你布局的代码和媒体查询的代码会更加直观。最后，用了命名的代码最终会生成与实现了相同布局但没有使用命名一样的代码。
