人脸关键点是对于图像中人脸五官与轮廓定位的标注，主要用来对人脸的关键位置，如脸廓、眉毛、眼睛、嘴唇进行定位，人脸关键点检测是人脸识别过程中重要的一步，同时这项技术在现实中也有极其广泛的应用空间：如表情识别、人脸动画合成等，尤其是最近几年被大家所熟知熟用的直播、美颜、特效相机等互联网应用产品的爆火也得益于人脸关键点相关技术水平的提升。



人脸关键点用到了哪些技术？


采用先进的“深度学习”算法，用神经网络模拟人类的大脑可大大提升计算机的视觉能力，使机器更“聪明”地识别到人脸。旷视于2012年开始使用深度学习并推出了 Face++ 人工智能开放平台，致力于为企业和开发者用户提供高效率、高精度的人脸识别相关能力，而在人脸关键点环节中都需要运用哪些人脸识别相关的技术呢？

1) “Detection”——人脸检测，检测到人脸后才能提取面部的关键点；

2) “Landmark”——人脸关键点检测，用来提取面部关键位置的关键点；

3) “Tracking”——人脸跟踪与人脸关键点跟踪，用来跟踪面部的运动。


Face++ 人工智能开放平台为用户开放了人脸检测 API、人脸分析 API 以及人脸关键点 SDK 。通过 Detection ，计算机可以检测图片或视频中的人脸，并对于检测到的每张人脸，返回其人脸框坐标以及 face_token。用户可以将 face_token 传给其他 API ( 如 Face Analyze API ) 以进行后续处理和分析。而对于最大的5个检测出的人脸，Detect API 还可以返回其 83 点人脸关键点和人脸属性。


和 API 接口在线调用不同， 旷视提供的人脸关键点 SDK 是支持在移动设备上离线运行 Face++ 人脸识别算法的开发包，可以实时追踪并完美贴合视频流中的所有人脸，并返回 106 个高精度人脸关键点。有了流畅和高精度的人脸关键点定位，用户才能够对应用中的人脸图像进行各类复杂变换操作，例如人脸妆容、滤镜贴纸和 3D 动画模型等。

如何区分关键点的性能



不同的应用需求对于关键点定位的能力有不同程度的要求，用户要用什么方式才能测试 SDK 的性能符不符合自己的应用需求呢？首先我们需要知道判别关键点性能优劣的四个维度：


1

贴合度高

好的关键点能够完美地贴合面部五官，即使在大角度或大动作的视频流中抖动，依然能保持稳定。相反若关键点的贴合能力不够则会大大影响产品的体验和应用效果，比如在美妆应用中唇膏和眼线等美妆特效会贴到真实的五官之外让用户一秒变“如花”。

2

稳定性强

好的关键点不会抖动而是稳稳地贴在面部。如果关键点的稳定性不强，用户在特效应用中为基于人脸的照片或视频加贴纸的时候，就会造成贴纸抖动。

3

跟踪速度快

好的关键点能实时跟踪每一帧画面，不会造成画面的卡顿、滞后，且人脸快速转头移动时也不会跟丢，相反性能不够好的 SDK 在进行人脸跟踪的时候比较滞后，会引起画面卡顿、占用大量CPU计算资源等情况。

4

鲁棒性强

好的关键点能在各种极端场景下保持稳定的质量，包括遮挡、大侧脸、运动、恶劣光照等，而鲁棒性差的关键点在大角度的侧脸、遮挡或较差的光照条件下就会丢失或错位。


因此，在选择算法和 SDK 的时候，用户至少需要对关键点相关 SDK 进行以下测试：


1

正面测试


主要测试关键点的贴合度、稳定性；

2

遮挡测试


遮挡面部部分区域，包括戴眼镜、遮嘴巴，测关键点是否稳定和贴合；

3

明暗测试


在不同光线条件下测试关键点是否稳定和贴合；

4

大角度测试


上下左右旋转头部，检测关键点的稳定和贴合；

5

多表情测试


不同姿态下不同面部表情，检测关键点的稳定和贴合。

还有哪些黑科技？



除了人脸检测、人脸跟踪和人脸关键点检测技术，旷视 Face++ 人工智能开放平台还在关键点的应用中加入了很多创新手段来增强关键点的鲁棒性。例如在市场中有些厂商通过模拟瞳孔点位来补全关键点，但点位却无法跟随眼珠转动，Face++ 则通过眼球跟踪技术可以避免这个问题从而让美瞳等特效更真实；为了解决在一些动态场景中特效跟随的问题，Face++ 利用 3D Pose 可实现直接对贴纸的三维旋转；而针对遮挡问题，Face++ 的关键点算法可以提供每个点被遮挡的置信度来为特征点定位提供参考值。
