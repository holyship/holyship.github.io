--- title: 快报 | Nginx在Web服务器市场份额达到33.3％，而Apache则低于50％ author: Nana 译 date: '2017-04-11 00:00:00 +0000' --- {% raw %}

导读：一半的网络仍然运行在Apache的Web服务器上，但有三分之一已经使用Nginx，差距正在快速缩小。 我们仔细看看详细的统计数据和趋势。


Nginx 是增长最快的网络服务器，市场份额达到 33.3％。 七年前，只有 3.9％。 平均来说，排名前 1000 万网站中，每 1 分钟都有一家新网站开始切换到 Nginx。 为了容易更好理解这个数字，nginx 增长速度是另一个快速增长的 Web 服务器 Node.js 站点增长数量的 70 倍。（ArchNotes 小编注：全栈工程师一脸懵逼，“发生什么了？”）


自 2010 年以来， Apache 的市场份额从 71.5％ 下降到刚刚低于 50％， 微软 IIS 从 20.6％ 下降到 11.3％。



正如预期的那样， Nginx 从 Apache 获得大部分市场份额，但是也有不少网站从 Nginx 迁移走的案例。网站管理员目前非常务实和灵活。


Nginx 是高流量网站中最流行的 Web 服务器，在全球前 10,000 个站点中的市场份额为 58.4％。 Apache 仍然是前百万网站领域内的领先者，但差距已经很小：Nginx 为 39.7％，Apache 为 42.8％。



Nginx 在俄罗斯拥有 76.8％ 的市场份额，这款软件原来是由 Igor Sysoev 于 2004 年开发的。在前苏联的一些国家，如白俄罗斯，乌克兰和哈萨克斯坦，达到了类似的市场份额。 Nginx 在亚洲，非洲和南美洲的一些国家也是市场的领导者。 Apache 在大多数国家，尤其是美洲和欧洲大部分地区仍然是保持领先，而 Microsoft-IIS 则在中国，土耳其，埃及和一些阿拉伯国家领先。

Nginx 是支持一些更现代协议领先的 Web 服务器，这可能是人们切换到它的原因之一。 支持 HTTP/2 的站点中有 76.8％ 使用 Nginx，而只有 2.3％ 的站点依赖于 Apache。 这并不奇怪，考虑到 Apache 的 HTTP/2 模块仍然处于“实验性”阶段。 同样，支持 IPv6 的所有站点的 63.3％ 运行在 Nginx 上，几乎是 Apache 的三倍。


Nginx 也是运行 Ruby ， Python 和 Scala 语言以及使用现代 JavaScript 库（如 AngularJS ， Vue,js 和 React）的网站之间的领先 Web 服务器。 在查看 Web 服务器的底层操作系统时，没有明确的趋势。 基于 Gentoo 的系统更喜欢 Nginx，而 Red Hat / CentOS 系统绝大多数运行 Apache。


预测未来市场份额趋势的一个很好的指标是新站点的技术选型。在这一点 Nginx 在所有网站上的表现都保持较好态势，但仅是微弱领先：Nginx 为 36.4％，Apache 为 47.7％（ArchNotes 小编：还是 apache 领先？），Microsoft-IIS 为 10.8％。 关注我们每日更新的趋势图将会为您呈现未来的实时发展。


请注意，该文章中提到的所有趋势和数字在撰写本文时（2017.4.10）均有效。 我们的调查经常更新，这些趋势和数字可能随着时间的推移而改变。
